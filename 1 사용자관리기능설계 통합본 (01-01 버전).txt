목적


관리자와 일반 근로자의 역할을 구분하고, 역할 기반 UI를 제공하여 사용자 경험을 최적화합니다.
사용자관리기능 설계 업무의 기준 문서로 활용합니다.


주요 기능


카카오톡 로그인
사용자 역할 구분(관리자 / 일반근로자)
역할 기반 UI 분리(웹 / 앱)
안전 관련 문서 취합 및 결재
위험요인 발굴 및 피드백
메시지 기능(전체 / 개별)
위험요인 개선 여부 확인 및 자동 보고서 생성
안전점수 평가 및 엑셀 출력
본사 / 현장 / 근로자 메뉴 구성
확장성 고려한 구조 설계


개발 전략


초기 MVP는 노코드 툴을 활용할 수 있으며, 필요에 따라 Flutter 등 개발툴을 사용할 수 있습니다.
향후 외부 저장소(서버, NAS 등) 연동이 가능하도록 개발합니다.


통합플랫폼 설계 및 유지관리 추천사항


플랫폼은 하나로 통합하되, 역할별로 기능과 UI를 모듈화하여 분리합니다.
권한 기반 접근 제어를 통해 근로자, 관리자, 본사 간의 기능을 명확히 구분합니다.
공통 인프라(데이터베이스, 인증, 알림 등)를 활용하여 유지보수 효율성을 높입니다.
초기 설계 문서(아키텍처 다이어그램, 역할별 기능 명세 등)를 문서화하여 확장성과 협업을 강화합니다.


사용자 기술 수준 고려


본 프로젝트는 비개발자(문외한)도 이해하고 운영할 수 있도록 설계됩니다.
모든 기능은 직관적 UI/UX와 자동화 중심으로 구성되며, 복잡한 설정은 최소화합니다.
문서, 보고서, 코드 주석 등은 비개발자 기준으로 작성됩니다.
개발자는 기술적 설명을 쉽게 풀어 전달하며, 협업자는 이를 기준으로 커뮤니케이션합니다.


코딩 폴더 세분화


모든 개발 관련 폴더는 기능 단위로 최대한 세분화하여 구성합니다.
예시: 로그인, 사용자관리, 위험요인보고, 리포트생성, 알림 등 기능별로 폴더를 나눕니다.
각 폴더에는 해당 기능의 코드, 테스트, 문서, UI 설계 기준을 포함합니다.
세분화된 구조는 유지보수, 협업, 확장성 측면에서 유리하며, 비개발자도 기능 흐름을 쉽게 파악할 수 있도록 합니다.
