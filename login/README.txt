[기능명] login

[기능 개요]
- 이 기능은 사용자관리 시스템의 시작점으로, 카카오톡 OAuth2.0 인증을 통해 사용자를 식별하고 역할을 분기합니다.
- 로그인 후 사용자 정보를 DB에 저장하며, JWT를 발급하여 세션을 유지합니다.
- 로그인 성공 시 사용자 역할에 따라 관리자/근로자 UI로 자동 이동합니다.

[기능 흐름]
1. 사용자가 로그인 버튼 클릭
2. 카카오 인증 페이지로 리디렉션
3. 사용자 정보 수신 (이름, 이메일, 카카오 ID 등)
4. DB에 사용자 정보 저장 또는 기존 정보 조회
5. 사용자 역할 확인 (관리자 / 근로자)
6. JWT 토큰 발급 및 세션 생성
7. 역할에 따라 UI 분기:
   - 관리자 → 관리자 대시보드
   - 근로자 → 근로자 홈 화면

[코딩 구조]
- login/
  ├─ kakao_auth.js : 카카오 인증 요청 및 콜백 처리
  ├─ token_handler.js : JWT 토큰 생성 및 검증
  ├─ user_mapper.js : 사용자 정보 DB 저장 및 조회
  ├─ role_router.js : 사용자 역할에 따른 UI 분기 처리
  └─ README.txt : 기능설계 문서

[참고]
- 대원칙: 사용자관리기능설계 통합본 (01-01 버전)
- 모든 코드는 비개발자 기준으로 주석과 흐름을 설명합니다.
- UI 설계는 역할 기반으로 분리되며, 로그인 후 자동 분기됩니다.
