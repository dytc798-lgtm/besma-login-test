[UI 설계 문서명] login_UI설계.txt

[화면 목록]
1. 로그인 화면 (공통)
2. 카카오 인증 페이지 (외부)
3. 역할 분기 화면
4. 관리자 UI (로그인 직후)
5. 근로자 UI (로그인 직후)

[화면 상세 설계]

1. 로그인 화면
- 구성 요소:
  - 로고 영역
  - "카카오톡으로 로그인" 버튼
  - 하단: 개인정보 처리방침 / 이용약관 링크
- 동작:
  - 버튼 클릭 시 카카오 OAuth 인증 페이지로 이동

2. 카카오 인증 페이지 (외부)
- 카카오톡 계정 선택 및 권한 동의
- 사용자 정보(이메일, 이름, 카카오 ID) 제공 동의

3. 역할 분기 화면
- 신규 사용자:
  - "환영합니다" 메시지
  - 기본 정보 입력(소속, 직무 등)
- 기존 사용자:
  - DB 조회 후 자동 역할 확인 → 다음 화면으로 이동

4. 관리자 UI (로그인 직후)
- 상단: 관리자 이름, 로그아웃 버튼
- 주요 메뉴:
  - 위험요인 보고 현황
  - 결재 대기 문서
  - 전체 메시지 발송
  - 안전점수 리포트
- 특징:
  - 카드형 대시보드
  - 알림 영역에 “신규 보고서” 표시

5. 근로자 UI (로그인 직후)
- 상단: 근로자 이름, 로그아웃 버튼
- 주요 메뉴:
  - 위험요인 보고하기
  - 받은 메시지 확인
  - 내 안전점수 확인
- 특징:
  - 버튼형 UI
  - "보고하기" 버튼을 가장 크게 배치

[UI 설계 기준]
- 역할 기반 분리: 관리자와 근로자는 로그인 이후 전혀 다른 UI로 이동
- 모바일/웹 공통성: 모바일은 버튼 중심, 웹은 카드형 대시보드 중심
- 비개발자 친화적: 메뉴 이름은 직관적 단어 사용
- 확장성 고려: 추후 본사 전용 메뉴 추가 가능
